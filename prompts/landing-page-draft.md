# Landing Page Draft Prompt

Use this prompt only with stored validation evidence.

Inputs:

- Idea record.
- Latest validation report.
- Stored query and autocomplete evidence.
- Stored pain quotes.
- Stored competitor and pricing evidence.
- Current score and missing proof.

Output a landing page draft only when the evidence justifies a payment-intent
test. If the evidence is weak, output `not_justified` and do not write
optimistic landing page copy.

Required sections:

- H1 using exact task language.
- Subheadline.
- Problem section.
- Before and after section.
- Three-step workflow.
- Preview-before-pay explanation.
- Trust and privacy section.
- One-time pricing hypothesis.
- FAQ.
- CTA variants.

Rules:

- Do not claim the business is validated.
- Do not invent features, volume, conversion rates, or customer proof.
- Do not use subscription framing unless stored evidence supports recurring use.
- Keep copy narrow and tied to evidence-backed query language.
